# 数据清洗与跨库去重规则

本研究以已筛定的 109 本高质量期刊石油领域文献数据为基础数据源，整合 CNKI、WOS 与 CSCD 三库记录，构建 2011-2025 年正式分析语料。研究对象为发文机构而非期刊，因此高质量期刊属性在本文中主要承担数据来源约束作用，而不作为后续评价对象。为保证语料构建过程可重复、可审查且可在论文中清晰表述，本文将数据清洗与跨库去重划分为“源内可用性清洗”与“跨库文献标识合并”两个阶段。

在源内清洗阶段，CNKI 数据不再以 DOI 是否存在作为入库门槛，而是以核心书目信息完整性作为准入标准。具体而言，仅当记录至少包含可用题名字段、发表时间字段、期刊字段和机构字段时，才纳入正式语料构建。题名可来自 `title_cn_en`、`title` 或 `title_en`；发表时间可来自 `publish_date` 或 `year`；期刊信息可来自 `journal_cn` 或 `journal_en`；机构信息可来自 `institution_extracted`、`institution` 或 `institution_norm`。这一处理的依据在于，CNKI 的 DOI 字段存在缺失、不规范及源内编号混入等情况，若将 DOI 设为强制条件，会无必要地剔除仍具书目识别价值的中文文献，不利于后续机构层面的全量分析。

在 DOI 规范化阶段，本文采用严格 DOI 识别规则，仅接受符合 `10.<prefix>/<suffix>` 结构的标准 DOI，并统一从 `doi` 与 `registered_doi` 两字段中提取标准 DOI 键。相较于宽松的 DOI 截取方式，严格规则能够避免将期刊号片段、源内编号或残缺字符串误识别为 DOI。进一步地，若同一数据库内部出现“同一严格 DOI 对应多个规范化题名”的情形，则将该 DOI 判定为不稳定 DOI，不作为自动合并依据，而是保留到审查表中人工核验。与此同时，诸如 `CNKI:...` 之类的源内原始 DOI 样式字符串不再被视为可靠书目标识，也不参与自动去重。

在跨库去重阶段，本文采用“双主键”合并策略。第一主键为可用标准 DOI 键；第二主键为元数据回退键 `meta_key`，由规范化题名、年份、第一作者和期刊四个字段共同构成。具体合并规则为：当两条记录共享同一可用标准 DOI 时，判定为同一文献；当记录缺乏可用标准 DOI 时，仅在其 `meta_key` 唯一对应既有分组且不存在 DOI 冲突的前提下，才允许执行元数据合并；若同一 `meta_key` 同时对应多个候选分组，则视为歧义匹配，不自动吞并。该规则的核心思想是优先使用高置信度标识符，在标识符不足时再使用较严格的书目组合键，以减少跨库误并风险。

完成分组合并后，本文采用来源感知的字段择优融合策略生成正式主表。中文题名、中文摘要和中文关键词优先保留 CNKI 或 CSCD 信息，英文题名、英文摘要、英文关键词和被引频次优先保留 WOS 信息，机构字段则保留用于后续机构标准化和国际合作分析的合并结果。同时，在正式主表中显式保留来源库组合、去重依据、标准 DOI 键、元数据去重键以及机构-国家配对字段，以支持后续结果追踪。

为保证方法透明性，本文不将 DOI 冲突记录和弱相似候选记录直接并入正式语料，而是单独导出审查表。审查表至少覆盖三类情形：其一，题名、年份和作者高度相似但缺乏可用标准 DOI 的弱相似记录；其二，元数据高度一致但 DOI 不一致的冲突记录；其三，同库内部 DOI 对应多个题名的不稳定 DOI 记录。正式主表、去重统计表、方法说明文件和审查表共同构成本文语料构建的可追溯证据链。上述规则保证了正式分析语料既具备较好的覆盖度，又能在论文中以清晰、可辩护的方式说明其科学依据与潜在限制。
