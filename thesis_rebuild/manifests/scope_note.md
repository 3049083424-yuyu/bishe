## 3. 去重方法边界

当前正式语料的跨库去重规则为：

1. 先以严格标准 DOI 作为高优先级主键，标准 DOI 统一从 `doi` 与 `registered_doi` 中提取，且仅接受符合 `10.<prefix>/<suffix>` 结构的字符串。
2. 若同一来源内部同一严格 DOI 对应多个规范化题名，则将其判定为不稳定 DOI，不纳入 DOI 自动合并。
3. 不再把 `CNKI:...` 这类源内原始 DOI 样式字符串视为可靠自动合并键。
4. 在缺乏可用标准 DOI 时，使用 `题名 + 年份 + 第一作者 + 期刊` 构造的严格元数据回退键。
5. DOI 冲突记录与弱相似记录不直接并入正式主表，而是下放审查表。

## 4. 历史口径边界

- `doi_required` 口径仅作为历史对照，不再写入正式结论、正式图表和正式摘要。

- 当前重建目标是形成“方法上可解释、结果上可追溯”的正式语料，而非机械复制旧链路的每一个中间状态。
